django standalone

앞선 포스팅에서 네이버 블로그에 있는 벤큐 피시방 목록을 크롤링 하는데에는 성공했다. 이제 크롤링한 정보를 db에 pcroom모델 객체로 저장해야 된다. pcroom의 모델은 다음과 같다.

manage.py shell 에서 하듯이 django orm을 이용해 pcroom 모델을 임의로 생성해보았다.

이제 script.py 를 실행하면 PCroom 객체가 하나 생성되고 db에 저장되야 하지만

대강 이런 에러가 뜬다. script.py에 장고 환경 변수가 설정이 안되있다는 뜻이다. 이를 해결하기 위해 django standalone이 있다. django standalone은 일반 파이썬 스크립트에서도 ORM 같은 장고 기능을 사용할 수 있게해준다. django standalone을 사용하기 위해서는 파일 상단에 다음 4줄만 추가해 주면 된다.

django standalone을 몰랐을 때는 sqlite db에 직접 접근해서 쿼리문으로 PCroom객체들을 저장했다. 잘 작동했지만 문제가 있었다. 배포할 때는 postgres같은 다른 db를 사용해야 하는데 sqlite 쿼리문으로는 데이터를 저장할 수 없는 것이다. django standalone은 외부 파일에서 ORM말고 다른 장고 기능을 사용할 때에도 요긴하게 쓸 수 있을것 같다.
